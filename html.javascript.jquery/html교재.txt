<1>html은 무엇인가
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
●[웹브라우저]라는 SW에서 [문자열], [이미지], [동영상], <[입력양식]> 등을 출력하기 위해 사용되는 언어를 말한다.
●HyperText Markup Language의 약어이다
●HTML은 실무적으로 [웹브라우저]가 [웹서버]에 접속한 후 [웹서버]가 [웹브라우저]쪽으로 던져주는 언어이다.
	HTML안에는 DB 연동 결과물이 내재되어 있다. 
●HTML은 프로그램은 언어는 아니다. 로직 같은 형식이 없다.
●<주의>개발자는 [입력양식] 태그를 매우 조심해야한다. 오라클 및 Spring과 연결되어 있음

<2>html 차원에서 태그(tag)는 무엇인가
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
●HTML 안에서 문자열, 이미지, 동영상, 입력양식 등을 출력하기 위해 사용되는 형식을 말한다.
-----------------------------------------------------------
●태그 (tag)의 형식1
-----------------------------------------------------------
	<태그명 속성명="속성값">
		컨텐츠
	</태그명>
	---------------------------------
	▶<태그명>과 </태그명> 사이의 컨텐츠가 태그 기능의 영향을 받는다
	---------------------------------
	<예> <font color="blue">Bigdata</font>	=> 문자열 Bigdata가 파랑색으로 웹브라우저에 출력됨
	<예> <b>Bigdata</b>			=> 문자열 Bigdata가 두꺼워져 웹브라우저에 출력됨
	<예> <center>Bigdata</center>		=> 문자열 Bigdata가 웹브라우저의 중앙위치로 출력됨
-----------------------------------------------------------
●태그 (tag)의 형식2
-----------------------------------------------------------
	<태그명 속성명="속성값"/>
	--------------------------------------
	<예> <hr/>	=>수평선 출력됨
	<예> <input type="text" name="uid" size="10"> => 한줄의 문자열을 입력하는 입력양식 출력
-----------------------------------------------------------
	▶태그명 --> 태그 종류명을 말한다.
	▶속성명 --> 태그의 성질을 말한다. 
	▶속성값 --> 속성명에 대한 값을 말한다.
	▶속성명="속성값" 은 0개 이상 나올 수 있다.
	▶속성값은 " 또는 ' 로 감싼다. 만약 속성값 중간에 공백이 없으면 " 또는 '는 생략할 수 있으나 되도록 사용한다.
	▶컨텐츠에는 문자열 또는 또 다른 태그가 나올 수 있다.
	▶태그명, 속성명은 대문자 또는 소문자를 써도 된다. 되도록 소문자를 사용한다.

<6>html에서 사용하는 색상값 표현 방법은 무엇인가?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■

<방법1>영문표기법
--------------------------------------------------	
	▶영문으로 색상을 표기하는 방법
	▶<예>black, blue, yellow, white, red, gray, green, purple, aqua, olive, silver, navy, 
		       fuchsia, lime, maroon, teal 등
--------------------------------------------------	
<방법2>RGB(RED GREEN BLUE) 표기법
--------------------------------------------------	
	▶Red 표현 16진수 2자리, Green 표현 16진수 2자리, Blue 표현 16진수 2자리를
	   붙여 색상을 표기하는 방법
	▶총 16,777,216 개의 색상을 지원
	▶<예>#000000(black), #FFFFFF(white), #FF0000(red), #00FF00(green), #0000FF(blue)
	▶실무적으로는 영문표기법 보다 RGB표기법을 더 많이 쓴다.

--------------------------------------------------------
절대단위
--------------------------------------------------------
	in, cm, mm.
	pt=>포인트. 1포인트=1/72인치.
	pc=>파이카. 1파이카=12포인트
--------------------------------------------------------
상대단위
--------------------------------------------------------
	%  	=> 웹화면의 좌우 또는 상하를 100%로 놓고 설정하는 단위.
	px 	=> 스크린의 1픽셀을 1로 함.
	em	=> 웹브라우저의 기본 폰트 크기(16px)을 1로 하는 단위
		     소수점 표현도 가능. 예>0.85em

<9>웹 개발 시 [웹디자이너]. [웹 퍼블리셔], [웹 프로그래머]의 역할이란
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■

웹 디자이너
	-- 포토샵, 일러스트레이터를 사용하여 웹화면에 출력되는 이미지의 그래픽작업을 한다.
웹 퍼블리셔
	-- HTML, CSS, Javascript를 사용하여 웹화면의 UI(=User Interface) 제작 작업을 한다.
	-- <참고>Javascript는 프로그래밍 언어의 한 종류이므로 [웹 퍼블리셔]는 프로그래머로서의 역량이 더 많이 필요하다고 볼 수 있다

웹 프로그래머
	-- 서버 쪽 언어(JSP, Java, PHP, ASP, .NET)을 사용하여 UI로부터 또는 UI로의 데이터의 이동, DB 연동 작업을 한다.
	-- 입력양식 관련 HTML은 개발자가 할 수도 있다.
	-- 복잡하고 어려운 Javascript 작업을 하는 경우도 있다.

<15>table 태그, td 태그, tr 태그가 뭐 할때 사용하는 것인가
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■

	문자열, 또는 이미지를 [행]과 [열]로 표시하는 태그다. 
	즉 컨텐츠의 배치를 관리하는 태그이다.
	-----------------------------------------------------
	n행 m열의 행과 열을 만들어 컨텐츠를 배치하려면 아래 형식을 가진다.
	-----------------------------------------------------
	<table>
		<caption>표제목</caption>
		<thead>
			<tr><th>해더1</th><th>헤더2</th>~<th>헤더m</th></tr>
		</thead>
		<tbody>
			<tr><td>데이터11</td><td>데이터12</td>~<td>데이터1m</td></tr>
			<tr><td>데이터21</td><td>데이터22</td>~<td>데이터2m</td></tr>
			<tr><td>데이터31</td><td>데이터32</td>~<td>데이터3m</td></tr>
			.....
			<td>데이터n1</td><td>데이터n2</td>~<td>데이터nm</td>
		</tbody>
	</table>
	-----------------------------------------------------
	●<tr>의 개수가 전체 행의 개수이다.
	●한 행의 <td> 또는 <th>의 개수가 열의 개수이다.
	●<caption> 태그는 생략이 가능하다.
	●<thead>,<tbody> 는 생략 가능하다. 그러나 jQuery 에서 테이블을 컨트롤 할 때는 유용하게 사용된다.
	●</td></th></tr> 는 생략 가능하나 영역을 나타내므로 되도록 써야 한다.
	-----------------------------------------------------
	●table 태그의 속성
		-- border		-> 표 전체의 선 두께값 설정. border을 생략하면 border="0"이 기본이다.
		-- bordercolor	-> 표 전체의 테두리 색 설정
		-- width		-> 표 전체의 너비 설정
		-- height		-> 표 전체의 높이 설정
		-- cellpadding	-> 각 셀 안의 문자와 경계선 사이 여백 설정
		-- cellspacing	-> 셀과 셀 사이의 튀어나온 부분의 너비 설정
		-- align		-> 표 전체가 화면에 좌중우 에 배치할 때 설정
		-- bgcolor	-> 표 전체의 배경색 설정
		-- background	-> 표 전체의 배경 이미지 설정
		-- style		-> 1개 이상의 CSS 설정 할때 사용한다.
				<table style="border-collapse:collapse;"> 
				테이블의 입체적 경계선을 무너뜨려 단일선으로 만든다
	------------------------------------------------------------------------------------------
	●tr 태그의 속성
		-- bgcolor	-> 한 행의 배경색 설정
		-- style		-> 1개 이상의 CSS 설정 할때 사용한다.
				<tr style="cursor:pointer"> 행에 마우스를 대면 손모양이 모인다.
		-- on이벤트명	-> 이벤트가 발생하면 원하는 자스코딩을 실행한다.
	------------------------------------------------------------------------------------------
	●td, th 태그 속성
		-- width		-> 셀의 너비 설정
		-- height		-> 셀의 높이 설정
		-- align		-> 셀 안의 문자열의 위치(왼쪽, 가운데, 오른쪽) 설정
				돈이 들어있는 열에 해당하는 셀은 보통 오른쪽(right)가 맞다
		-- valign		-> 셀 안의 문자열의 위치(상중하) 설정
		-- bgcolor	-> 셀의 배경색 설정
		-- background	-> 셀의 배경 이미지 설정
		-- colspan	-> 지정한 개수의 좌우 셀을 합친다
		-- rowspan 	-> 지정한 개수의 위아래 셀을 합친다

<16>입력 양식 관련 태그 종류
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
◆ [데이터]를 [입력] 또는 [선택]할 수 있는 태그 또는 그와 관련된 태그를 지칭
◆ 해당 태그에 입력 또는 선택한 데이터는 서버의 웹프로그래밍 언어로 전달되고 최종 DB(=오라클)에 저장된다.
-----------------------------------------
◆ 입력 양식 관련 태그의 종류
-----------------------------------------
-----------------------------------------
form
-----------------------------------------
	▶ 여러 [입력 양식]을 1개의 그룹으로 묶는 태그. 택배 상자에 비유.
	▶ 서버의 URL주소, 데이터 전송방법 등을 설정할 수 있다.
	▶ 실질적인 입력양식은 아니나 중요한 태그.
	------------------------------------------
	▶ form 태그의 속성(주의, 체크할 것)
	------------------------------------------
		- name 	→ form 태그의 이름을 설정하는 속성명. form 태그가 여러 있을 경우 구별하기위해 주는 속성.
		- action	→ form 태그 안의 입력양식에 입력(선택)된 데이터를 보낼 서버 쪽 URL 주소를 설정하는 속성.
		- method	→ 입력(선택) 데이터를 서버로 보내는 방법을 설정 (주의, 체크하기)
				<예> method="get" 	=> 입력(선택) 데이터가 URL 뒤에 붙어 전송
							     보안성이 낮은 데이터 전송에 사용
							     이 방식으로 데이터 전송 시 URL 주소에 전송 데이터를 삽입만
							     하면 되므로 코딩 수고가 줄어든다.
							     method 속성 설정이 생략되면 method="get"가 디폴트이다.
				<예> method="post" 	=> 입력(선택) 데이터가 안보이게 전송
							     보안성이 높은 데이터 전송에 사용
							     이 방식으로 데이터 전송 시 입력양식에 데이터를
							     삽입해야 되므로 코딩 수고가 늘어난다. 즉, 입력양식이 꼭 필요하다.
		- enctype → 전송될 데이터의 형식을 설정. 문자를 보낼때 또는 파일을 보낼 때 이 설정으로 해야 함.
			     만약 입력 양식 중에 파일업로드가 있으면 다음 처럼 enctype 속성값을 준다.
			     -----------------------------------------------------
			     enctype="multipart/form-data" (외울것)
			     -----------------------------------------------------
		<주의>-	     enctype="multipart/form-data" 를 생략하면 파일이 전송되지 않고 파일의 이름만 전송된다.
		<주의>-	     파일업로드가 없으면 enctype="multipart/form-data"를 생략하면 된다.
		target	→ form 태그 안의 입력양식에 입력(선택)된 데이터를 보낼 때 어느 화면을 이용해 보낼지 설정.
			   target 속성이 생략되면 form 태그가 있는 현재 화면을 이용하여 웹서버로 전송한다.
-----------------------------------------
input
-----------------------------------------
	▶ 데이터를 입력 또는 선택(=체크)할 수 있는 [입력 양식]을 출력하는 태그.
	▶ input 태그의 속성
		- type="text"  	→ 한 줄의 [문자열]을 입력받는 입력양식을 설정.
		- type="passward"	→ 한 줄의 [암호문자열]을 입력받는 입력양식을 설정.
		- type="checkbox"	→ [체크박스] 입력 양식 설정. 다중 체크 가능. 얘만 등장하면 관련된 모든 프로그래밍은 복잡해 진다.
				    DB의 테이블구조도 복잡해진다.<개발자 최대의 난제>
		- type="radio"	→ [라디오버튼] 입력 양식 설정. 다중 체크 불가능.
		- type="file"	→ [파일박스] 양식 설정. 파일 업로드에 사용. 얘만 등장하면 관련된 모든 프로그래밍은 복잡해진다.
				    파일 업로드 사용 시 보안성에 문자가 생길 가능성이 있다. 해커 주요 침투 경로.
				    이 입력 양식이 있으면 반드시 form 태그에는 enctype="multipart/form-data"가 있어야 한다.
		- type="hidden"	→ 화면에 보이지 않고 값만 저장하는 양식 설정.
				   화면에 보이고 싶지 않지만 서버로 데이터를 보내고 싶을 때 사용.
				   검색화면에서 많이 사용되며
				   개발자가 조심해야 하는 입력양식 중 하나이다.
				   <팁> 개발과정에서는 type="hidden"을 type="text"로 해놓고 개발하고 개발 완료 후
				   type="hidden"로 바꿔버린다.
				   <pk값이 주로 저장되는 양식>
		================================================================================
		- type="button"	→ [버튼] 양식 설정. 버튼을 블릭하면 벌어지는 일은 개발자가 코딩해야 함. 사용자 정의 버튼.
		- type="submit"	→ [서버 전송 버튼] 양식 설정. 같은 폼 소속의 데이터를 서버쪽으로 보내는 전용버튼이다.
		- type="reset"	→ [초기화 버튼] 양식 설정. 같은 폼 소속의 입력양식 데이터를 초기화하는 전용 버튼이다. 
		================================================================================
		- name		→ 입력 양식 이름을 지정하는 속성. 주로 [오라클]의 [컬럼명]과 일치 시킨다.
		- value		→ 입력 또는 선택한 값이 저장되는 속성. 서버의 웹프로그래밍 언어로 전달되고 최종 DB의 테이블에 저장.
				   <주의>type="button", type="submit", type="reset"의 경우 버튼에 삽입되는 단순 문자열 설정이고
				    서버의 웹프로그래밍 언어로 전달되지도 않는다.
		================================================================================
		- size		→ 입력 양식의 좌우 길이를 설정.
				   <주의> type="text", type="passward" 에서는 좌우 길이지만 select 태그 안에서는 펼쳐놓는 개수를 말한다.
		================================================================================
		- maxlength	→ 키보드로 입력 가능한 문자열의 개수 설정.

				   <주의>입력한 데이터가 한글일 경우 브라우저마다 maxlength 속성이 안 먹힐 수 있다.

웹브라우저에서 발생하는 모든 움직임을 말한다.
이벤트는 주로 유저에 의해 발생되고 이벤트는 웹서버에게 일종의 명령을 내리는 것과 동일하다.
즉, 이벤트는 웹브라우저와 웹서버간의 소통 역할을 한다.
웹브라우저에서 발생하는 이벤트는 웹브라우저가 감지하고 있다.
----------------------------------------------------------
웹브라우저 감지할 수 있는 이벤트의 종류
----------------------------------------------------------
	click 		이벤트 → 마우스 왼쪽 버튼을 클릭하는 행위이다. 제일 많이 사용하는 이벤트.
	mouseover 	이벤트 → 글씨나 이미지에 마우스를 갖다 대는 행위이다.
	mouseout	이벤트 → 글씨나 이미지에 마우스르 갖다 댄 후 때는 행위이다.
	mousedow	이벤트 → 마우스에 달려 있는 임의 마우스 버튼을 클릭하는 행위이다.
	mousedragged	이벤트 → 마우스를 드래그하는 행위이다.
	---------------------------------------------------------------------------------------------------------
	keydown		이벤트 → 키보드를 누르는 행위이다. 누르고 있을 때 한번만 이벤트가 발생한다.
	keypress		이벤트 → 키보드를 누르는 행위이다. 누르고 있을 때 계속 이벤트가 발생한다.
	keyup		이벤트 → 키보드를 누른 후 때는 행위이다.
	select		이벤트 → 문자열을 드래그해서 선택하는 행위이다.
	change		이벤트 → <select~> 태그라면 목록을 바꿔 선택하는 행위이다.<주의>
				<input type="checkbox" ~> 또는 <input type="radio" ~> 에서는
				체크가 들어가거나 체크가 빠지는 이벤트이다.
				<input type="text" ~> 또는 <input type="passward" ~> 또는 <textarea~>
				에서는 입력 내용이 기존과 다른 후 커서가 빠져나올때 발생하는 이벤트이다.
	---------------------------------------------------------------------------------------------------------
	load 		이벤트 → <body> 태그 안의 모든 내용을 읽어 화면에 출력(=로딩)한 후의 이벤트이다.
	unload		이벤트 → 현재 보여지는 웹페이지가 웹브라우저 화면에서 사라질 때 발생하는 이벤트이다.
	---------------------------------------------------------------------------------------------------------
	기타 등등 이벤트
